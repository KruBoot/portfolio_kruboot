# คู่มือสอนนักเรียนสร้าง GitHub Portfolio Online ด้วยเว็บเบราว์เซอร์เท่านั้น

คู่มือนี้ออกแบบสำหรับนักเรียน ม.4-ม.6 ที่เริ่มจากศูนย์ ไม่เคยใช้ GitHub มาก่อน และต้องการสร้าง Portfolio Online แบบง่าย ๆ ที่เปิดดูบนหน้า GitHub ได้ทันที

แนวทางนี้ไม่ใช้ Terminal, ไม่ใช้คำสั่ง Git, ไม่ต้องติดตั้งโปรแกรมเพิ่ม ใช้เพียงเว็บเบราว์เซอร์ เช่น Chrome, Edge, Firefox หรือ Safari

## เป้าหมายเมื่อเรียนจบ

เมื่อทำตามคู่มือนี้จนจบ นักเรียนจะสามารถ

- สมัครและเข้าใช้งาน GitHub ได้
- สร้าง repository สำหรับ Portfolio ของตนเองได้
- สร้างไฟล์และโฟลเดอร์บน GitHub ผ่านเว็บได้
- เขียนไฟล์ Markdown เบื้องต้นได้
- จัดทำสารบัญ Portfolio ที่คลิกลิงก์ไปยังหน้าต่าง ๆ ได้
- เพิ่มประวัติ ผลงาน กิจกรรม เกียรติบัตร Reflection และ TCAS Summary ได้
- ดูประวัติการแก้ไขงานผ่าน Commit history ได้
- ส่งลิงก์ Portfolio ให้ครู เพื่อน หรือมหาวิทยาลัยดูได้

## ภาพรวมสิ่งที่จะสร้าง

นักเรียนจะสร้าง Portfolio รูปแบบนี้

```text
student-portfolio-example/
├── README.md
├── ABOUT-ME.md
├── GOAL-MAP.md
├── projects/
│   ├── PROJECT-01-peer-support-campaign.md
│   ├── PROJECT-02-short-video-mental-health.md
│   └── PROJECT-03-reading-journal.md
├── activities/
│   └── ACTIVITY-LOG.md
├── certificates/
│   └── CERTIFICATES.md
├── reflections/
│   └── REFLECTION.md
├── TCAS-SUMMARY.md
└── assets/
    └── README.md
```

และจะมีไฟล์ `README.md` หน้าแรกของ repository เป็นสารบัญหลักสำหรับเปิดดูบน GitHub

## คำศัพท์พื้นฐานแบบง่าย

| คำศัพท์ | ความหมายแบบนักเรียน |
|---|---|
| GitHub | เว็บไซต์สำหรับเก็บงานออนไลน์ ดูประวัติการแก้ไข และแชร์ผลงาน |
| Repository หรือ repo | โฟลเดอร์งานหลัก 1 ชุด เช่น Portfolio ของเรา |
| README.md | หน้าแรกที่ GitHub แสดงให้อ่านทันที |
| Markdown | วิธีเขียนข้อความให้มีหัวข้อ ตาราง ลิงก์ และรายการ โดยไม่ต้องทำเว็บ |
| Commit | การบันทึกการเปลี่ยนแปลง 1 ครั้ง พร้อมข้อความอธิบาย |
| History | ประวัติการแก้ไขงานย้อนหลัง |
| Public | ทุกคนที่มีลิงก์สามารถเปิดดูได้ |
| Private | เฉพาะเจ้าของหรือคนที่ได้รับเชิญเท่านั้นที่ดูได้ |

## ก่อนเริ่มเรียน

นักเรียนควรเตรียมสิ่งต่อไปนี้

- อีเมลที่ใช้งานได้จริง
- เว็บเบราว์เซอร์
- อินเทอร์เน็ต
- ชื่อ Portfolio ที่ต้องการใช้ เช่น `my-student-portfolio`
- ข้อมูลของตนเองแบบย่อ เช่น ชื่อ ระดับชั้น โรงเรียน ความสนใจ
- รายการผลงานหรือกิจกรรม 2-3 ชิ้น
- รูปภาพหรือเกียรติบัตร ถ้ามี

ข้อควรระวัง: อย่าใส่ข้อมูลส่วนตัวที่ละเอียดเกินไป เช่น เลขบัตรประชาชน ที่อยู่จริง เบอร์โทรศัพท์ส่วนตัว หรือข้อมูลของคนอื่นโดยไม่ได้รับอนุญาต

## บทที่ 1: สมัคร GitHub

1. เปิดเว็บเบราว์เซอร์
2. ไปที่ `https://github.com`
3. กดปุ่ม `Sign up`
4. กรอกอีเมล
5. ตั้งรหัสผ่าน
6. ตั้ง username เป็นภาษาอังกฤษ เช่น `ploy-portfolio` หรือ `napatsorn-example`
7. ทำตามขั้นตอนยืนยันอีเมล
8. เมื่อล็อกอินสำเร็จ ให้จำ username ของตนเองไว้

ถ้านักเรียนมีบัญชี GitHub อยู่แล้ว ให้กด `Sign in` แทน

## บทที่ 2: สร้าง Repository ใหม่

1. หลังจากเข้าสู่ระบบ GitHub แล้ว มองหาปุ่ม `+` มุมขวาบน
2. คลิก `New repository`
3. ช่อง `Repository name` ให้ตั้งชื่อ เช่น `student-portfolio`
4. ช่องคำอธิบายจะใส่หรือไม่ใส่ก็ได้ เช่น `My online portfolio for TCAS`
5. เลือก `Public` ถ้าต้องการให้คนที่มีลิงก์เปิดดูได้
6. ติ๊ก `Add a README file`
7. กด `Create repository`

เมื่อสร้างสำเร็จ GitHub จะพาไปหน้า repository ใหม่ และจะเห็นไฟล์ `README.md`

## บทที่ 3: เข้าใจหน้า Repository

บนหน้า repository จะเห็นส่วนสำคัญดังนี้

- ชื่อ repository อยู่ด้านบน
- รายชื่อไฟล์อยู่ตรงกลาง
- ไฟล์ `README.md` แสดงเป็นหน้าอ่านอยู่ด้านล่าง
- ปุ่ม `Add file` ใช้เพิ่มไฟล์ใหม่หรืออัปโหลดไฟล์
- ปุ่มรูปดินสอ ใช้แก้ไขไฟล์
- แท็บ `Commits` หรือเลข commit ใช้ดูประวัติการแก้ไข

ให้จำง่าย ๆ ว่า GitHub คือแฟ้มงานออนไลน์ และ `README.md` คือหน้าปกของแฟ้ม

## บทที่ 4: แก้ไข README หน้าแรกให้เป็นสารบัญ

1. คลิกไฟล์ `README.md`
2. กดไอคอนรูปดินสอ `Edit this file`
3. ลบข้อความเดิมออก
4. ใส่ตัวอย่างนี้ลงไป

```markdown
# Portfolio Online ของฉัน

สวัสดีค่ะ/ครับ นี่คือแฟ้มสะสมผลงานออนไลน์ของฉัน

## สารบัญ

| ลำดับ | หน้า | ใช้ทำอะไร |
|---|---|---|
| 1 | [แนะนำตัว](student-portfolio-example/ABOUT-ME.md) | ประวัติ ความสนใจ และทักษะ |
| 2 | [แผนเป้าหมาย](student-portfolio-example/GOAL-MAP.md) | เป้าหมายการเรียนต่อ |
| 3 | [ผลงาน](student-portfolio-example/projects/PROJECT-01-peer-support-campaign.md) | ตัวอย่างผลงานเด่น |
| 4 | [บันทึกกิจกรรม](student-portfolio-example/activities/ACTIVITY-LOG.md) | Log กิจกรรมและบทบาท |
| 5 | [TCAS Summary](student-portfolio-example/TCAS-SUMMARY.md) | สรุปสำหรับยื่นมหาวิทยาลัย |
```

5. เลื่อนลงมาด้านล่าง
6. ช่อง commit message ให้เขียนว่า `Create portfolio table of contents`
7. เลือก `Commit directly to the main branch`
8. กด `Commit changes`

นี่คือการบันทึกงานครั้งแรกด้วย commit

## บทที่ 5: สร้างโฟลเดอร์และไฟล์บน GitHub

GitHub ไม่มีปุ่มสร้างโฟลเดอร์ตรง ๆ แต่เราสร้างโฟลเดอร์ได้โดยพิมพ์ชื่อไฟล์พร้อมเครื่องหมาย `/`

ตัวอย่าง ถ้าต้องการสร้างไฟล์

```text
student-portfolio-example/ABOUT-ME.md
```

GitHub จะสร้างโฟลเดอร์ `student-portfolio-example` และไฟล์ `ABOUT-ME.md` ให้เอง

วิธีทำ

1. กลับไปหน้าแรกของ repository
2. กด `Add file`
3. เลือก `Create new file`
4. ที่ช่อง `Name your file...` ให้พิมพ์ชื่อไฟล์พร้อมโฟลเดอร์ เช่น `student-portfolio-example/ABOUT-ME.md`
5. เขียนเนื้อหาในช่องใหญ่
6. ใส่ commit message
7. กด `Commit changes`

ให้ทำซ้ำกับไฟล์อื่น ๆ ทีละไฟล์

## บทที่ 15: อัปโหลดรูปภาพหรือไฟล์ PDF ผ่านเว็บ

1. เข้าไปที่โฟลเดอร์ `student-portfolio-example/assets`
2. กด `Add file`
3. เลือก `Upload files`
4. ลากไฟล์รูปภาพหรือ PDF ลงไป
5. รอให้อัปโหลดเสร็จ
6. ใส่ commit message เช่น `Upload portfolio evidence`
7. กด `Commit changes`

ข้อแนะนำการตั้งชื่อไฟล์

- ใช้ภาษาอังกฤษตัวพิมพ์เล็ก
- ใช้ `-` คั่นคำ
- หลีกเลี่ยงชื่อเช่น `IMG_1234.jpg`
- ตัวอย่างที่ดี: `peer-support-poster.jpg`

## บทที่ 16: ใส่รูปภาพใน Markdown

ถ้าไฟล์ Markdown อยู่หน้าแรกของ repository และรูปอยู่ใน assets ให้เขียนแบบนี้

```markdown
![คำอธิบายรูป](student-portfolio-example/assets/peer-support-poster.jpg)
```

ถ้าไฟล์ Markdown อยู่ในโฟลเดอร์ `projects/` และต้องการเรียกรูปจาก `assets/` ให้เขียนแบบนี้

```markdown
![คำอธิบายรูป](../assets/peer-support-poster.jpg)
```

ถ้ารูปไม่ขึ้น ให้ตรวจว่า

- ชื่อไฟล์ตรงกันหรือไม่
- ตัวพิมพ์เล็กตัวพิมพ์ใหญ่ตรงกันหรือไม่
- ตำแหน่งโฟลเดอร์ถูกต้องหรือไม่

## บทที่ 17: ดู Preview ก่อนบันทึก

ตอนแก้ไขไฟล์ GitHub จะมีแท็บ `Edit` และ `Preview`

1. เขียนเนื้อหาในแท็บ `Edit`
2. กดแท็บ `Preview`
3. ตรวจว่าหัวข้อ ตาราง ลิงก์ และรูปภาพแสดงถูกต้อง
4. ถ้าถูกต้องแล้วค่อยกด `Commit changes`

นิสัยที่ดี: ดู Preview ทุกครั้งก่อน commit

## บทที่ 18: ดูประวัติการแก้ไขงาน

วิธีดู history ของทั้ง repository

1. กลับไปหน้าแรกของ repository
2. มองหาคำว่า `Commits`
3. คลิกเข้าไป
4. จะเห็นรายการ commit เรียงจากใหม่ไปเก่า
5. คลิก commit แต่ละอันเพื่อดูว่าแก้ไฟล์อะไรบ้าง

วิธีดู history ของไฟล์เดียว

1. เปิดไฟล์ที่ต้องการ เช่น `ABOUT-ME.md`
2. กดปุ่ม `History`
3. ดูว่าไฟล์นี้เคยแก้ไขกี่ครั้ง

นี่คือจุดเด่นของ GitHub เพราะครูสามารถเห็นพัฒนาการของงาน ไม่ใช่เห็นแค่ไฟล์สุดท้าย

## บทที่ 19: วิธีเขียน Commit message ให้สวย

Commit message ควรสั้น ชัดเจน และบอกว่าทำอะไร

ตัวอย่างที่ดี

- `Create portfolio table of contents`
- `Add about me page`
- `Add first project`
- `Update activity log`
- `Add reflection after project`
- `Upload certificates`

ตัวอย่างที่ควรหลีกเลี่ยง

- `update`
- `แก้`
- `final`
- `final2`
- `จริงๆfinal`

## บทที่ 20: ตรวจงานก่อนส่ง

ให้ตรวจตาม checklist นี้

- [ ] หน้าแรก `README.md` มีสารบัญ
- [ ] ลิงก์ในสารบัญกดได้จริง
- [ ] มีหน้า About Me
- [ ] มี Goal Map
- [ ] มีผลงานอย่างน้อย 2-3 ชิ้น
- [ ] มี Activity Log
- [ ] มี Certificates หรือรายการหลักฐาน
- [ ] มี Reflection
- [ ] มี TCAS Summary
- [ ] ไม่มีข้อมูลส่วนตัวที่เสี่ยงเกินไป
- [ ] รูปภาพหรือไฟล์หลักฐานเปิดดูได้
- [ ] Commit history อ่านรู้เรื่อง

## บทที่ 21: วิธีส่งลิงก์ Portfolio

1. เปิดหน้า repository ของตนเอง
2. คัดลอก URL จากช่อง address bar ของเว็บเบราว์เซอร์
3. ส่งลิงก์ให้ครูหรือผู้รับผลงาน

ตัวอย่างลิงก์

```text
https://github.com/username/student-portfolio
```

ถ้า repository เป็น `Private` คนอื่นจะเปิดดูไม่ได้ จนกว่าจะได้รับสิทธิ์ หรือเปลี่ยนเป็น `Public`

## บทที่ 22: ถ้าต้องการเปลี่ยน Private เป็น Public

1. เข้า repository
2. กด `Settings`
3. เลื่อนลงไปบริเวณ `Danger Zone`
4. เลือกเปลี่ยน visibility
5. ทำตามขั้นตอนยืนยันของ GitHub

ก่อนเปลี่ยนเป็น Public ต้องตรวจข้อมูลส่วนตัวให้รอบคอบ

## บทที่ 23: แผนการสอนแบบ 4 คาบ

### คาบที่ 1: รู้จัก GitHub และสร้าง repo

เป้าหมาย: นักเรียนสมัคร GitHub สร้าง repository และแก้ README หน้าแรกได้

งานท้ายคาบ:

- มี repository ของตัวเอง
- มี README ที่เป็นสารบัญเบื้องต้น
- มี commit อย่างน้อย 1 ครั้ง

### คาบที่ 2: สร้าง About Me และ Goal Map

เป้าหมาย: นักเรียนเริ่มเล่าเรื่องตัวเองและเป้าหมายการเรียนต่อ

งานท้ายคาบ:

- มี `ABOUT-ME.md`
- มี `GOAL-MAP.md`
- ลิงก์จาก README ไปยังสองไฟล์นี้ได้

### คาบที่ 3: เพิ่ม Projects และ Activity Log

เป้าหมาย: นักเรียนเขียนผลงานโดยเน้นบทบาทและสิ่งที่ได้เรียนรู้

งานท้ายคาบ:

- มีผลงานอย่างน้อย 1 ชิ้น
- มี Activity Log
- มี commit message ที่อ่านรู้เรื่อง

### คาบที่ 4: เพิ่ม Reflection, TCAS Summary และตรวจส่ง

เป้าหมาย: นักเรียนเชื่อมโยงผลงานกับคณะที่สนใจ และตรวจความพร้อมก่อนส่ง

งานท้ายคาบ:

- มี Reflection
- มี TCAS Summary
- ตรวจ checklist ก่อนส่งครบ
- ส่งลิงก์ repository ให้ครู

## ปัญหาที่พบบ่อยและวิธีแก้

### กดลิงก์แล้วขึ้น 404

สาเหตุที่เป็นไปได้

- ชื่อไฟล์ผิด
- โฟลเดอร์ผิด
- ใช้ตัวพิมพ์ใหญ่เล็กไม่ตรงกัน

วิธีแก้: เปิดรายชื่อไฟล์จริงใน GitHub แล้วคัดลอกชื่อไฟล์ให้ตรง

### ตาราง Markdown ไม่ขึ้นเป็นตาราง

สาเหตุที่เป็นไปได้

- ลืมบรรทัด `|---|---|`
- จำนวนช่องในแต่ละแถวไม่เท่ากัน

วิธีแก้: กด Preview แล้วตรวจทีละแถว

### รูปภาพไม่ขึ้น

สาเหตุที่เป็นไปได้

- path รูปผิด
- ไฟล์ยังไม่ได้อัปโหลด
- ชื่อไฟล์ไม่ตรง

วิธีแก้: คลิกเข้าโฟลเดอร์ assets แล้วตรวจชื่อไฟล์จริง

### นักเรียนเผลอใส่ข้อมูลส่วนตัวมากเกินไป

วิธีแก้

1. กดแก้ไขไฟล์
2. ลบข้อมูลที่ไม่ควรเผยแพร่
3. Commit การแก้ไข
4. ถ้าข้อมูลอ่อนไหวมาก ควรปรึกษาครู เพราะข้อมูลอาจยังอยู่ใน history

## สูตรเขียน Portfolio ให้ไม่เว่อร์ แต่ดูจริง

ให้ใช้โครงนี้ทุกครั้งที่เขียนผลงาน

```text
ฉันทำอะไร
ทำไมถึงทำ
ฉันรับผิดชอบส่วนไหน
ผลลัพธ์คืออะไร
ฉันได้เรียนรู้อะไร
เกี่ยวข้องกับคณะที่สนใจอย่างไร
```

ตัวอย่างประโยคที่ดี

```text
กิจกรรมนี้ทำให้ฉันเห็นว่าการรับฟังเพื่อนไม่ใช่การรีบให้คำแนะนำ แต่คือการให้พื้นที่กับความรู้สึกของอีกฝ่าย และรู้ว่าเมื่อใดควรขอความช่วยเหลือจากครูหรือผู้ใหญ่
```

## งานสุดท้ายสำหรับนักเรียน

ให้นักเรียนส่งสิ่งต่อไปนี้

- ลิงก์ GitHub repository
- ชื่อ-นามสกุล
- ห้องเรียน
- คณะที่สนใจ
- ผลงานที่ภูมิใจที่สุด 1 ชิ้น
- สิ่งที่ได้เรียนรู้จากการทำ Portfolio บน GitHub 3 ข้อ

## สรุป

การทำ Portfolio บน GitHub ไม่ได้มีไว้สำหรับนักเรียนสายโปรแกรมเมอร์เท่านั้น แต่นักเรียนทุกสายสามารถใช้ GitHub เป็นแฟ้มสะสมผลงานออนไลน์ได้ เพราะช่วยให้จัดระบบงาน แสดงหลักฐาน เขียน Reflection และดูพัฒนาการผ่าน history ได้

สิ่งสำคัญที่สุดไม่ใช่ความสวยงามของหน้าเว็บ แต่คือความจริงใจ ความชัดเจน และการอธิบายให้เห็นว่าเราเติบโตจากประสบการณ์ของตนเองอย่างไร
